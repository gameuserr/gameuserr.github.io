大声协议

#大声协议#

#目标：

1982年，Clayton Lewis向用户体验专业人士介绍了一种思维协议。该协议已针对游戏行业进行了修改; 它是为研究人员和游戏设计师制作“数据丰富”结果的最有效方法之一。

#过程：

在思考中，玩家在游戏过程中口头表达思想，感受，行为和经历。研究人员进行会议，在必要时探测玩家以大声分享经验。
这是一项实验室研究，因此邀请参与者进入实验室。通常进行测试直到结果饱和，即没有学到新的东西，或者时间或金钱耗尽。

数据收集： 在朗读会话中收集的数据是关于用户所说的内容以及用户在游戏会话中所说的内容的注释（手写，音频录制或视频录制）。

数据分析： 然后对这些数据进行分析，以制定解决参与者在游戏中交互过程中遇到的可用性问题的建议。

#Resources
C.H. Lewis. (1982). Using the “Thinking Aloud” Method in Cognitive Interface Design,Tech. report RC 9265, IBM.
C. Wright and A. Monk. (1991).
The Use of Think-Aloud Evaluation Methods in Design,ACM SIGCHI Bull., vol. 23, no. 1, pp. 55–57.

#欢迎来到游戏用户研究网站

在过去的十年中，游戏产业从一个小生态市场演变为娱乐业的主要部门。此外，游戏已经成为流行媒体的核心形式，影响了青少年和成年人的生活。这种影响力要求游戏行业研究更多的创新设计，并增加更多种类的互动，以适应所有年龄段。这引发了游戏用户研究（GUR）领域。GUR已发展成为一个社区，作为国际游戏开发者协会特殊利益集团，每年举办一次国际峰会和研讨会（见gamesuserresearch.org）。

游戏用户研究人员帮助游戏设计师改善玩家体验，以满足设计师的目标，最重要的是，让游戏的可玩性变得更好。为了识别和理解玩家体验，这些研究人员采用了各种方法，包括定性和定量数据收集和分析，借鉴社会科学，心理学，人机交互等等。

该网站整理了一些最重要的GUR方法和工具。
